Oto opis logiki aplikacji.

Szablony:

1. Szablony dzielimy na 4 typy: Ticker, Belka, zegar i obrazek (przemianujmy pozniej na grafika)
2. Szablony składaja się z kodu HTML CSS i Javascript
3 Szablony można swobodnie wgfrywać do aplikacji i pobierać z aplikacji. 
4. tworząc nową grafikę w banku grafik wybieramy szablon, który zawiera domyślny, przykładowy wygląd danego typu grafiki
5. Jeśli wyedytuje szablon to chcę zapisać moje zmieny. pozwól też taki wyedytowany szablon pobierac i instalowac w aplikacji tak jak reszte innych.

Bank grafik:

1. zawiewra elementy graficzne tworzone na podstawie szablonów
2. elementy te mozna grupowac w grupy - celem wspólnego wejścia na output.
3. elementy banku grafik są edytowalne w UI inspektora. Kazdy typ danego elementu (belka, ticker) powinny mieć dostosowane UI do swoich ustawień, funkcji i mozliwosci.
4. elementy banku grafik można przenosić drag and drop.

baza danych:

1 proszę o optymalizację bazy danych. pozwól zgrywac z UI plik db.json oraz instalować wcześniej zgrany z poziomu aplikacji.
